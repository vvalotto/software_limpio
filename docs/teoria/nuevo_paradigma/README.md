# El Nuevo Paradigma

> *"Si las máquinas escriben, nosotros podemos pensar, diseñar, decidir, evaluar y dirigir."*

---

## El Cambio de Era

Programar —escribir código— se ha vuelto operativo. Necesario, pero ya no suficiente.

Las máquinas pueden generar código más rápido y con menos errores sintácticos. Esto no es una amenaza: **es una liberación**.

### Evidencia del Cambio

| Fuente | Hallazgo |
|--------|----------|
| Peng et al. (2023) | Tareas completadas 55.8% más rápido con Copilot |
| GitHub Research (2024) | +5% aprobación en code reviews |
| DORA Report (2024) | +2.1% productividad, pero -7.2% estabilidad |

### La Paradoja DORA

El tiempo ahorrado por IA se absorbe en tareas de bajo valor si no hay formación sólida en principios de diseño (**Vacuum Hypothesis**).

## Contenidos

### [El Rol Profesional](rol_profesional.md)

El profesional del siglo XXI no es quien más código produce, sino quien mejor comprende:
- Qué debe construirse
- Por qué debe construirse así
- Cómo saber si se construyó correctamente

**Las cuatro competencias:**

| Competencia | Descripción |
|-------------|-------------|
| **Dirigir** | Proporcionar contexto, restricciones y criterios a la IA |
| **Evaluar** | Medir la calidad del código con herramientas objetivas |
| **Refinar** | Iterar usando las métricas como feedback |
| **Decidir** | Tomar decisiones de diseño y asumir responsabilidad |

### [El Triángulo de Competencias](triangulo_competencias.md)

La integración de tres ejes que la literatura trata separadamente:

```
           PRINCIPIOS
          (qué y por qué)
               ▲
              ╱ ╲
             ╱   ╲
            ╱     ╲
           ╱       ╲
          ╱         ╲
    MÉTRICAS ◄────► IA
  (verificación)  (herramienta)
```

**Flujo de trabajo:**
1. Comprender el principio
2. Generar código con IA
3. Medir con herramientas
4. Comparar contra umbrales
5. Iterar hasta cumplir
6. Documentar decisiones

## El Modelo de Formación

### Modelo Obsolescente

```
ENFOQUE TRADICIONAL
───────────────────
    Aprender a PROGRAMAR
           ↓
    Aprender FRAMEWORKS
           ↓
    (quizás) Aprender PATRONES
           ↓
    Producir código
```

**Problema:** La IA está demoliendo este modelo desde la base.

### Modelo Propuesto

```
ENFOQUE ÉTICO
─────────────
    Fundamentos FILOSÓFICOS
    (virtudes, responsabilidad, propósito)
           ↓
    Principios de DISEÑO LIMPIO
    (los 6 fundamentos universales)
           ↓
    Verificación con MÉTRICAS
    (objetividad, umbrales, feedback)
           ↓
    IA como HERRAMIENTA
    (dirigir, evaluar, decidir)
```

## Comparativa

| Aspecto | Formación Tradicional | Enfoque Propuesto |
|---------|----------------------|-------------------|
| Foco | Sintaxis, lenguajes, frameworks | Principios, ética, métricas |
| Rol de la IA | Amenaza o ayuda marginal | Herramienta central dirigida |
| Verificación | Subjetiva (code review) | Objetiva (métricas con umbrales) |
| Fundamento | Técnico | Filosófico-ético |
| Paradigma | Atado a OO o FP | Agnóstico, universal |
| Profesional | Productor de código | Director de calidad |

## Posicionamiento Único

Mientras el mundo investiga cómo enseñar programación básica con IA, este proyecto lidera en:

- **Formación de criterio profesional de calidad**
- **Integración sistemática principios-métricas-IA**
- **Redefinición del rol: de programador a evaluador/decisor**
- **Verificación objetiva de diseño en era IA**
